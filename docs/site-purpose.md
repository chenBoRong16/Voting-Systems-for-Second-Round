# 網站目的

本專案的核心是「Top4 兩輪」設計：
第一輪用簡單多數（只看第一志願）先選出 Top4，
第二輪再用排序選制（IRV / Borda / Ranked Pairs / Benham / Minimax）決定最終勝者。

這系列網站的目的是讓一般人也能看懂：流程是什麼、限制是什麼、不同選制會造成什麼差異。

若總候選人數 <= 4 人，第一輪只是在「原封不動保留全部候選人」，沒有實際篩選作用，
因此可直接跳過第一輪，進入第二輪。
第一輪的目的，是在候選人過多時先縮減人數，
避免選票無法設計成單排/單張、難以人工計票而被迫集中或改成電子計票，
進而增加對選制的不信任。

本專案採用「多個子站」的方式呈現內容：

- 一部分子站負責 **教育與說明**（讓大眾與專家快速理解概念與限制）
- 一部分子站負責 **互動測試**（讓使用者調整偏好分布，觀察不同選制的輸出差異）

## 整體目標

本專案的網站希望能作為「現行單輪簡單多數（只看第一志願）」替代方案的參考材料，提供：

1. 可讀的解釋（非政治系背景也能懂）
2. 可重現的例子（同一組票型分布，切換選制對比結果）
3. 可討論的介面（把抽象規則轉成可看見的結算過程）

## 站點分工

- **我的選票設計**：
  - 介紹選票格式（含單排設計、第一偏好高亮等）
  - 說明它為何能簡化人工計票/查找，以及限制在哪裡

- **網站目的**（本頁）：
  - 解釋這系列網站在做什麼、不是在做什麼
  - 釐清名詞：第一輪/第二輪、初選/決選、排序票的意義

- **選制說明**：
  - 每個選制至少包含：直覺、計票步驟、優缺點、適用情境與常見誤解

- **第二輪測試網站**：
  - 提供互動工具，讓使用者直接輸入「第二輪候選人集合」的排序票分布
  - 切換不同選制，看結算過程與結果差異

## 重要聲明：第二輪測試站不包含第一輪

「第二輪測試網站」是刻意把第一輪排除的。原因是：

- 若把第一輪初選也塞進同一頁，會把「候選人篩選」與「決選規則」混在一起，反而更難討論
- 本站希望能針對 **同一組候選人集合**，比較不同第二輪選制的行為

若你想了解「Top4 兩輪」的完整想法，重點如下：

- 第一輪：用簡單多數（只看第一志願）先選出 Top4
- 第二輪：在 Top4 固定候選人集合下，使用排序選制決定最終勝者
- 本站的「第二輪測試網站」就是為了比較不同排序選制在同一組候選人下的結果差異
