# Top4 兩輪構想（方法論）

本專案常用的討論框架是「兩輪」：

1. **第一輪（初選/初輪）**：用簡單規則先把候選人縮小到 Top4
2. **第二輪（決選/終輪）**：只針對這個候選人集合，使用排序選制決定勝者

若總候選人數 <= 4 人，第一輪只是在「原封不動保留全部候選人」，沒有實際篩選作用，
因此可直接跳過第一輪，進入第二輪。
第一輪的目的，是在候選人過多時先縮減人數，
避免選票無法設計成單排/單張、難以人工計票而被迫集中或改成電子計票，
進而增加對選制的不信任。

## 為什麼把第一輪與第二輪拆開？

拆開有兩個好處：

- **討論更清楚**：第一輪在做的是「候選人篩選」，第二輪在做的是「勝者決定」。兩者目的不同。
- **比較更公平**：若候選人集合固定，我們才能把差異歸因到第二輪選制本身，而不是候選人先後被篩掉造成的差異。

## 第一輪：簡單多數（Plurality / FPTP）示意

這裡的「第一輪簡單多數」指：

- 每張票只看第一志願
- 依票數高低取 Top4

> 注意：這只是其中一種候選人篩選方式。本專案把它當作容易理解、容易實作的 baseline。

## 第二輪：只測試決選規則

本專案的互動工具（`/web/round2/`）是刻意設計成：

- 使用者 **直接輸入第二輪候選人集合** 的完整排序票分布
- 切換不同第二輪選制（IRV / Ranked Pairs / Borda / Benham / Minimax…）
- 觀察同一份分布在不同規則下，會導出什麼結果與過程

因此：

- `/web/round2/` **不包含第一輪初選**
- 若你想把兩輪串起來，請把第一輪產生的候選人集合當作第二輪的輸入前提
